## skill-security-auditor

> **Claude Code スキル向けの MVP セキュリティ監査ツール — 自身の盲点を発見して修正するよう設計されています。**

### 問題背景

Claude Code スキルは、Claude にシェルコマンドの実行、外部 API の呼び出し、ファイルの読み書き、認証情報の処理を指示する Markdown ファイルです。悪意のある、または不適切に書かれたスキルは、データを外部に漏洩させたり、任意のコードを実行したり、API キーを静かに漏らしたりする可能性があります。現在、スキルをロードする前にリスクを評価する標準的な方法はありません。

### このスキルの機能

`skill-security-auditor` は `~/.claude/skills/` ディレクトリ内のすべてのスキルに対して静的解析を実行します。7つのリスク次元でスコアリングし、構造化されたレポートを生成し、追記専用の監査ログを維持し、最も重要な点として — 自身の検出ロジックの欠陥を記録して継続的に改善します。

---

### 7次元リスクスコアリングマトリクス

各次元は **0〜10** でスコアリングされ、最終スコア **0〜100** に結合されます。

| # | 次元 | 重み | 検出内容 |
|---|------|------|---------|
| D1 | **ネットワーク露出** | 20% | 外部 HTTP 呼び出し、動的 URL 構築、生ソケット |
| D2 | **認証情報アクセス** | 20% | API キー、トークン、`.env` ファイル、キーチェーンアクセス |
| D3 | **コード実行** | 18% | `subprocess`、`eval`、`exec`、`sudo`、パイプ実行 |
| D4 | **ファイルシステムアクセス** | 15% | ワークスペース外の読み書き、`~/.ssh`、`~/.aws` へのアクセス |
| D5 | **データ漏洩** | 12% | 外部に送信される会話データ、Base64 エンコードされたペイロード |
| D6 | **依存関係リスク** | 8% | `git+` URL、バージョン未固定、非 PyPI インデックス |
| D7 | **プロンプトインジェクション面** | 7% | サニタイズなしでプロンプトに挿入される外部コンテンツ |

**リスクレベル：**

| スコア | レベル | アクション |
|--------|--------|-----------|
| 0–19 | 🟢 低 | 対応不要 |
| 20–39 | 🟡 中 | 30日以内に確認 |
| 40–59 | 🟠 高 | 7日以内に確認 |
| 60–79 | 🔴 重大 | レビュー保留のため隔離 |
| 80–100 | ⛔ ブロック | ロード禁止；人間の承認が必要 |

```
│  • スコアラー校正         • レポート生成                   │
└─────────────────────────────────────────────────────────┘
```

### 自己改善ループ

これがコア設計原則です：**監査ツールは自身の使用を通じて改善されます。**

各スキャン後、`risk_scorer.py` は `self_notes` を出力します — 自身の検出品質に関する構造化された観察結果です。これらのノートは監査ログに書き込まれ、レポートに表示されます。次のイテレーションサイクルでは、スコアラーは自身のノートを読み込み、具体的な修正案を提案します — 変更を適用する前に**人間の確認を待ちます**。

```
スキャン → self_notes → 改善提案 → 人間の確認 → パッチ適用 → 再スキャン → 検証
```

**自己発見によるバージョン履歴：**

| バージョン | トリガー | 修正内容 |
|-----------|---------|---------|
| v0.1.0 | 初期 | 7次元静的スキャナー |
| v0.2.0 | Self-note: D1 のヒットがコードブロック内 | ` ``` ` ブロックを除外；D6 `>=` の誤検知修正；D7 再構築 |
| v0.2.1 | `cosmos-policy` の自動散文解析 | D2 `token` パターンが広すぎた — ML 語彙（"tokenizer"）に誤マッチ |

### 使い方

```bash
git clone https://github.com/WilliamHe-cyber/claude-skills.git
cp -r claude-skills/skill-security-auditor ~/.claude/skills/
```

```
/security-audit                      # すべてのスキルをスキャン
/security-audit langchain            # 1つのスキルをスキャン
/security-audit --log                # 監査ログサマリーを表示
/security-audit --iterate            # 自己改善サイクルを実行
/security-audit --pre new-skill      # インストール前の事前確認
```

---

### 自動週次監査

このリポジトリは毎**月曜日 9:00 AM** に自動 Claude Code エージェントを実行します。全スキルをスキャンし、回帰を検出し、レポートを生成し — `self_notes` がある場合は改善案を一覧表示して、コードを変更する前に**人間の確認を待ちます**。

監査レポートは公開・バージョン管理されています：[`skill-security-auditor/reports/`](./skill-security-auditor/reports/)

---

### コントリビューション

1. **スキルを追加する** — `your-skill/SKILL.md` を作成して PR を開いてください。CRITICAL または BLOCKED のスコアを持つスキルはマージされません。
2. **スキャナーを改善する** — 誤検知や検出漏れを説明する Issue を開いてください。
3. **校正ノート** — 重みを調整する際は、`scoring_matrix.md` に `## Calibration Note` エントリを追加してください。
